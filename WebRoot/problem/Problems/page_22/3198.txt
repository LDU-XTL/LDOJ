</p><div class="ptx" lang="en-US"><span lang="en-us"><p>Imagine an infinite table with rows and columns numbered using the natural numbers. The following figure shows a procedure to traverse such a table assigning a consecutive natural number to each table cell:</p><div align="center"><img src="images/3198_1.gif"></div><p>This enumeration of cells can be used to represent complex data types using natural numbers:</p><ul><li><p>A pair of natural numbers (<i>i</i>, <i>j</i>) is represented by the number corresponding to the cell in row <i>i</i> and column <i>j</i>. For instance, the pair (3, 2) is represented by the natural number 17; this fact is noted by <i>P</i><sub>2</sub>(3, 2) = 17.</p></li><li><p>The pair representation can be used to represent <i>n</i>-tuples. A triplet (<i>a</i>, <i>b</i>, <i>c</i>) is represented by <i>P</i><sub>2</sub>(<i>a</i>, <i>P</i><sub>2</sub>(<i>b</i>, <i>c</i>)). A 4-tuple (<i>a</i>, <i>b</i>, <i>c</i>, <i>d</i>) is represented by <i>P</i><sub>2</sub>(<i>a</i>, <i>P</i><sub>2</sub>(<i>b</i>, <i>P</i><sub>2</sub>(<i>c</i>, <i>d</i>))). This procedure can be generalized for an arbitrary <i>n</i>:</p><p align="center"><i>P<sub>n</sub></i>(<i>a</i><sub>1</sub>, &hellip;, <i>a<sub>n</sub></i>) = <i>P</i><sub>2</sub>(<i>a</i><sub>1</sub>, <i>P<sub>n</sub></i><sub>&minus;1</sub>(<i>a</i><sub>2</sub>, &hellip;, <i>a<sub>n</sub></i>)),</p><p>where <i>P<sub>n</sub></i> denotes the <i>n</i>-tuple representation function, <i>n</i> &ge; 2. For example <i>P</i><sub>3</sub>(2, 0, 1) = 12.</p></li><li><p>A list of arbitrary length &lsaquo;<i>a</i><sub>1</sub>, &hellip;, <i>a<sub>n</sub></i>&rsaquo; is represented by</p><p align="center"><i>L</i>(&lsaquo;<i>a</i><sub>1</sub>, &hellip;, <i>a<sub>n</sub></i>&rsaquo;) = <i>P</i><sub>2</sub>(<i>n</i>, <i>P<sub>n</sub></i>(<i>a</i><sub>1</sub>, &hellip;, <i>a<sub>n</sub></i>)).</p><p>For example, <i>L</i>(&lsaquo;0, 1&rsaquo;) = 12.</p></li></ul><p>The Association of Convex Makers (ACM) uses this clever enumeration scheme in a polygon representation system. The system can represent a polygon, defined by integer coordinates, using a natural number as follows: given a polygon defined by a vertex sequence &lsaquo;(<i>x</i><sub>1</sub>, <i>y</i><sub>1</sub>), &hellip;, (<i>x<sub>n</sub></i>, <i>y<sub>n</sub></i>)&rsaquo; assign the natural number:</p><p align="center">L(&lsaquo;<i>P</i><sub>2</sub>(<i>x</i><sub>1</sub>, <i>y</i><sub>1</sub>), &hellip;, <i>P</i><sub>2</sub>(<i>x<sub>n</sub></i>, <i>y<sub>n</sub></i>)&rsaquo;).</p><p>ACM needs a program that, given a natural numbers that represents a polygon, calculates the area of the polygon. It is guaranteed that the given polygon is a simple one, i.e. its sides do not intersect.</p><p>As an example of the problem, the triangle with vertices at <code>(1,1)</code>, <code> (2,0)</code> and <code> (0,0)</code> is codified with the number <code>2141</code>. The area of this triangle is <code>1</code>.</p></span></div><p class="pst">Input</p><div class="ptx" lang="en-US"><p>The input consists of several test cases. Each test case is given in a single line of the input by a natural number representing a polygon. The end of the test cases is indicated with <code>*</code>.</p></div><p class="pst">Output</p><div class="ptx" lang="en-US"><p>One line per test case, preserving the input order. Each output line contains a decimal number telling the area of the corresponding encoded polygon. Areas must be printed with 1 decimal place, truncating less significative decimal places.</p></div><p class="pst">Sample Input</p><pre class="sio">2141
206
157895330
*</pre><p class="pst">Sample Output</p><pre class="sio">1.0
0.5
1.0</pre><p class="pst">
