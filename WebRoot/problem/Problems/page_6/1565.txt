</p><div class="ptx" lang="en-US">When a number is expressed in decimal, the kth digit represents a multiple of 10<sup>k</sup>. (Digits are numbered from right to left, where the least significant digit is number 0.) For example, 
<br>81307(10) = 8 * 10^4 + 1 * 10 ^3 + 3 * 10^2 + 0 * 10^1 + 7 * 10^0
<br>= 80000 + 1000 + 300 + 0 + 7
<br>= 81307.
<br>
<br>When a number is expressed in binary, the kth digit represents a multiple of 2^k . For example,
<br>
<br>10011(2) = 1 * 2^4 + 0 * 2^3 + 0 * 2^2 + 1 * 2^1 + 1 * 2^0
<br>= 16 + 0 + 0 + 2 + 1
<br>= 19.
<br>
<br>In skew binary, the kth digit represents a multiple of 2^(k+1)-1. The only possible digits are 0 and 1, except that the least-significant nonzero digit can be a 2. For example,
<br>
<br>10120(skew) = 1 * (2^5-1) + 0 * (2^4-1) + 1 * (2^3-1) + 2 * (2^2-1) + 0 * (2^1-1)
<br>= 31 + 0 + 7 + 6 + 0
<br>= 44.
<br>
<br>The first 10 numbers in skew binary are 0, 1, 2, 10, 11, 12, 20, 100, 101, and 102. (Skew binary is useful in some applications because it is possible to add 1 with at most one carry. However, this has nothing to do with the current problem.)
<br>
