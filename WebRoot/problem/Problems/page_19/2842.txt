</p><div class="ptx" lang="en-US">Let us consider an n-dimension matching problem of two matrices. Here the index number of each dimension of a matrix starts at 1. For two n-dimension matrixes S and T, if there is a position (p<sub>1</sub>, p<sub>2</sub>, p<sub>3</sub>, p<sub>4</sub>, …,p<sub>n</sub>) which satisfies that each element at the position (t<sub>1</sub>, t<sub>2</sub>, t<sub>3</sub>, t<sub>4</sub>, …,t<sub>n</sub>) in T is the same as the element at the position (p<sub>1</sub> + t<sub>1</sub> - 1, p<sub>2</sub> + t<sub>2</sub> - 1, p<sub>3</sub> + t<sub>3</sub> - 1, p<sub>4</sub> + t<sub>4</sub> - 1, …,p<sub>n</sub> + t<sub>n</sub> - 1) in S, we call it’s a matching position. So the n-dimension problem is to compute the matching position for given S and T. You can assume that the traditional string matching problem is the 1-dimension version of this problem, and the normal matrix matching problem is the 2-dimension version.</div><p class="pst">Input</p><div class="ptx" lang="en-US">The first line contains the positive number n, which is not larger than 10.
<br>The second line contains n positive numbers a<sub>1</sub>, a<sub>2</sub>, a<sub>3</sub>, ...,a<sub>n</sub>, representing the range of each dimension of matrix S.
<br>The third line contains n positive numbers b<sub>1</sub>, b<sub>2</sub>, b<sub>3</sub>, ...,b<sub>n</sub>, representing the range of dimensions of matrix T. 
<br>The fourth line gives the elements in S.
<br>The fifth line gives the elements in T.
<br>
<br>To give out an n-dimension matrix with size c<sub>1</sub> * c<sub>2</sub> * c<sub>3</sub> * ... * c<sub>n</sub> in a line, we will give the element at the position (d<sub>1</sub>, d<sub>2</sub>, d<sub>3</sub>, …,d<sub>n</sub>) in the matrix as the (c<sub>2</sub> * c<sub>3</sub> * ... * c<sub>n</sub> * (d<sub>1</sub> – 1) + c<sub>3</sub> * c<sub>4</sub> * ... * c<sub>n</sub> * (d<sub>2</sub> – 1) + ... + c<sub>n</sub> * (d<sub>n - 1</sub> – 1) + d<sub>n</sub>)–th element in the line.
<br>
<br>We guarantee that b<sub>i</sub> <= a<sub>i</sub>, elements in S and T are all positive integers that are not larger 100, and the number of the elements in S and T are both not larger than 500000.
<br></div><p class="pst">Output</p><div class="ptx" lang="en-US">You should output n numbers in a line, p<sub>1</sub>, p<sub>2</sub>, p<sub>3</sub>, …,p<sub>n</sub>, representing the matching position. Please print the lexically smallest one if there are many. We guarantee that there is at least one matching position.</div><p class="pst">Sample Input</p><pre class="sio">2
