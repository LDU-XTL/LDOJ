</p><div class="ptx" lang="en-US"><div><p>WhatNext Software creates sequence generators that they hope will produce fairly random sequences of 16-bit unsigned integers in the range 0&#8211;65535. In general a sequence is specified by integers <i>A</i>, <i>B</i>, <i>C</i>, and <i>S</i>, where 1 &#8804; <i>A</i> &lt; 32768, 0 &#8804; <i>B</i> &lt; 65536, 2 &#8804; <i>C</i> &lt; 65536, and 0 &#8804; <i>S</i> &lt; <i>C</i>. <i>S</i> is the first element (the <dfn>seed</dfn>) of the sequence, and each later element is generated from the previous element. If <i>X</i> is an element of the sequence, then the next element is</p><blockquote>(<i>A</i> * <i>X</i> + <i>B</i>) % <i>C</i> </blockquote><p>where '%' is the remainder or modulus operation. Although every element of the sequence will be a 16-bit unsigned integer less than 65536, the intermediate result <i>A</i> * <i>X</i> + B may be larger, so calculations should be done with a 32-bit <i>int</i> rather than a 16-bit <i>short</i> to ensure accurate results.</p><p>Some values of the parameters produce better sequences than others. The most embarrassing sequences to WhatNext Software are ones that never change one or more bits. A bit that never changes throughout the sequence is <dfn>persistent</dfn>. Ideally, a sequence will have no persistent bits. Your job is to test a sequence and determine which bits are persistent.</p><p>For example, a particularly bad choice is <i>A</i> = 2, <i>B</i> = 5, <i>C</i> = 18, and <i>S</i> = 3. It produces the sequence 3, (2*3+5)%18 = 11, (2*11+5)%18 = 9, (2*9+5)%18 = 5, (2*5+5)%18 = 15, (2*15+5)%18 = 17, then (2*17+5)%18 = 3 again, and we're back at the beginning. So the sequence repeats the the same six values over and over:</p><table style="text-align: left; width: 200px;" border="1" cellpadding="2" cellspacing="2"><tbody><tr><td style="font-family: monospace;" valign="middle" align="center">Decimal</td><td style="font-family: monospace;" valign="middle" align="center">16-Bit Binary</td></tr><tr><td style="font-family: monospace;" valign="middle" align="center">3</td><td style="font-family: monospace;" valign="middle" align="center">0000000000000011</td></tr><tr><td style="font-family: monospace;" valign="middle" align="center">11</td><td style="font-family: monospace;" valign="middle" align="center">0000000000001011</td></tr><tr><td style="font-family: monospace;" valign="middle" align="center">9</td><td style="font-family: monospace;" valign="middle" align="center">0000000000001001</td></tr><tr><td style="font-family: monospace;" valign="middle" align="center">5</td><td style="font-family: monospace;" valign="middle" align="center">0000000000000101</td></tr><tr><td style="font-family: monospace;" valign="middle" align="center">15</td><td style="font-family: monospace;" valign="middle" align="center">0000000000001111</td></tr><tr><td style="font-family: monospace;" valign="middle" align="center">17</td><td style="font-family: monospace;" valign="middle" align="center">0000000000010001</td></tr><tr><td valign="middle" align="center">overall</td><td style="font-family: monospace;" valign="middle" align="center">00000000000????1</td></tr></tbody></table><p>The last line of the table indicates which bit positions are always 0, always 1, or take on both values in the sequence. Note that 12 of the 16 bits are persistent. (Good random sequences will have no persistent bits, but the converse is not necessarily true. For example, the sequence defined by <i>A</i> = 1, <i>B</i> = 1, <i>C</i> = 64000, and S = 0 has no persistent bits, but it's also not random: it just counts from 0 to 63999 before repeating.) &nbsp;Note that a sequence does not need to return to the seed: with <i>A</i> = 2, <i>B</i> = 0, <i>C</i> = 16, and <i>S</i> = 2, the sequence goes 2, 4, 8, 0, 0, 0, ....</p></div></div><p class="pst">Input</p><div class="ptx" lang="en-US"><p>There are from one to sixteen datasets followed by a line containing only 0. Each dataset is a line containing decimal integer values for A, B, C, and S, separated by single blanks.</p></div><p class="pst">Output</p><div class="ptx" lang="en-US"><p>There is one line of output for each data set, each containing 16 characters, either '1', '0', or '?' for each of the 16 bits in order, with the most significant bit first, with '1' indicating the corresponding bit is always 1, '0' meaning the corresponding bit is always 0, and '?' indicating the bit takes on values of both 0 and 1 in the sequence.</p></div><p class="pst">Sample Input</p><pre class="sio">2 5 18 3
