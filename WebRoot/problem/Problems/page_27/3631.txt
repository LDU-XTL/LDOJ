</p><div class="ptx" lang="en-US"><div><p>One of the most fundamental data structure problems is the dictionary problem: given a set <i>D</i> of words you want to be able to quickly determine if any given query string <i>q</i> is present in the dictionary <i>D</i> or not. Hashing is a well-known solution for the problem. The idea is to create a function <i>h</i> : Σ<sup>* </sup>→ [0..<i>n</i>-1] from all strings to the integer range 0, 1, .., <i>n</i>-1, i.e. you describe a fast deterministic program which takes a string as input and outputs an integer between 0 and <i>n</i>-1. Next you allocate an empty hash table <i>T</i> of size <i>n</i> and for each word <i>w</i> in <i>D</i>, you set <i>T</i>[<i>h</i>(<i>w</i>)] = <i>w</i>. Thus, given a query string <i>q</i>, you only need to calculate <i>h</i>(<i>q</i>) and see if <i>T</i>[<i>h</i>(<i>q</i>)] equals <i>q</i>, to determine if <i>q</i> is in the dictionary. Seems simple enough, but aren't we forgetting something? Of course, what if two words in <i>D</i> map to the same location in the table? This phenomenon, called collision, happens fairly often (remember the Birthday paradox: in a class of 24 pupils there is more than 50% chance that two of them share birthday). On average you will only be able to put roughly √<i>n</i>-sized dictionaries into the table without getting collisions, quite poor space usage!</p><p>A stronger variant is Cuckoo Hashing. The idea is to use two hash functions <i>h</i><sub>1</sub> and <i>h</i><sub>2</sub>. Thus each string maps to two positions in the table. A query string <i>q</i> is now handled as follows: you compute both <i>h</i><sub>1</sub>(<i>q</i>) and <i>h</i><sub>2</sub>(<i>q</i>), and if <i>T</i>[<i>h</i><sub>1</sub>(<i>q</i>)] = <i>q</i>, or <i>T</i>[<i>h</i><sub>2</sub>(<i>q</i>)] = <i>q</i>, you conclude that <i>q</i> is in <i>D</i>. The name &quot;Cuckoo Hashing&quot; stems from the process of creating the table. Initially you have an empty table. You iterate over the words <i>d</i> in <i>D</i>, and insert them one by one. If <i>T</i>[<i>h</i><sub>1</sub>(<i>d</i>)] is free, you set <i>T</i>[<i>h</i><sub>1</sub>(<i>d</i>)] = <i>d</i>. Otherwise if <i>T</i>[<i>h</i><sub>2</sub>(<i>d</i>)] is free, you set <i>T</i>[<i>h</i><sub>2</sub>(<i>d</i>)] = <i>d</i>. If both are occupied however, just like the cuckoo with other birds' eggs, you evict the word <i>r</i> in <i>T</i>[<i>h</i><sub>1</sub>(<i>d</i>)] and set <i>T</i>[<i>h</i><sub>1</sub>(<i>d</i>)] = <i>d</i>. Next you put <i>r</i> back into the table in its alternative place (and if that entry was already occupied you evict that word and move it to its alternative place, and so on). Of course, we may end up in an infinite loop here, in which case we need to rebuild the table with other choices of hash functions. The good news is that this will not happen with great probability even if <i>D</i> contains up to <i>n</i>/2 words!</p></div></div><p class="pst">Input</p><div class="ptx" lang="en-US"><p>On the first line of input is a single positive integer 1 &le; <i>t</i> &le; 50 specifying the number of test cases to follow. Each test case begins with two positive integers 1 &le; <i>m</i> &le; <i>n</i> &le; 10000 on a line of itself, <i>m</i> telling the number of words in the dictionary and <i>n</i> the size of the hash table in the test case. Next follow <i>m</i> lines of which the <i>i</i>th describes the <i>i</i>th word <i>d<sub>i</sub></i> in the dictionary <i>D</i> by two non negative integers <i>h</i><sub>1</sub>(<i>d<sub>i</sub></i>) and <i>h</i><sub>2</sub>(<i>d<sub>i</sub></i>) less than <i>n</i> giving the two hash function values of the word <i>d<sub>i</sub></i>. The two values may be identical.</p></div><p class="pst">Output</p><div class="ptx" lang="en-US"><p><br>For each test case there should be exactly one line of output either containing the string &quot;successful hashing&quot; if it is possible to insert all words in the given order into the table, or the string &quot;rehash necessary&quot; if it is impossible.</p></div><p class="pst">Sample Input</p><pre class="sio">2
